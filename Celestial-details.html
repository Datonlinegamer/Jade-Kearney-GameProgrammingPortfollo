    Code Breakdown

    This section of code explains how the character interaction system works:

    • UInteractionComponent handles line tracing and sets CurrentFocusedActor each frame.
    • Actor receives OnBeginFocus or OnEndFocus when looked at or not.
    • When the player presses the interact key, UInteractionManager:
      • Checks if the actor is interactable.
      • Calls OnInteract() on the object.
    • This decouples responsibilities:
      • Player doesn't handle interaction logic.
      • Component handles detection.
      • Manager executes the interaction.

    What This Interface Does:

    This interface defines the standard contract that all interactable objects must follow. It allows any actor in the world to respond to player input in a consistent and modular way. Here's what each function does:

    • OnInteract() – Core logic triggered when a player interacts.
    • OnBeginFocus() – Called when the player starts looking at the object.
    • OnEndFocus() – Called when the player stops looking at the object.
    • CanInteract() – Logic to enable/disable interaction based on conditions.
    • GetInteractableName() – Returns a UI display name.
